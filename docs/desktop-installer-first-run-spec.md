# EchoMuse 桌面安装包与首次启动向导方案（实施规格）

更新时间：2026-02-25  
状态：待实施（方案已确认，按阶段落地）

## 1. 目标（你要的最终体验）

给普通用户一个接近“安装后就能用”的桌面体验：

1. 下载并运行 Windows 安装包（可选安装路径）
2. 安装完成后桌面出现图标（图标来源：项目根目录 `图1.png`）
3. 首次启动进入向导，而不是直接要求用户手动装一堆依赖
4. 向导先问用户用法模式，再根据选择安装依赖/模型：
   - `仅 API`
   - `仅本地模型`
   - `混合模式（本地 + API）`
5. 用户勾选本地模型时，支持多选，并且每个模型单独配置 `Flash / Thinking`
6. UI 文案不再写死 `Qwen Flash / Thinking`，而是跟随当前模型显示

## 2. 核心原则（避免再踩坑）

### 2.1 安装包保持轻量（必须）

安装包默认 **不打包** 以下大体积运行时/模型资源：

- `vendor/ollama/**/*`
- `vendor/whisper/**/*`
- `vendor/**/models/**/*`
- 其他大模型文件（`.bin` / `.gguf` 等）

原因：

- 安装包体积会暴涨（容易导致 NSIS 构建失败）
- 用户不一定需要本地模型
- 用户可能只想用 API
- 用户硬件不一定带得动你预置的模型

### 2.2 安装器只做“安装”，向导做“配置”

- `NSIS 安装器`：负责安装路径、桌面图标、开始菜单、卸载入口
- `首次启动向导（App 内）`：负责环境检查、模式选择、依赖安装、模型勾选、Provider 配置

这样稳定、可维护，也符合你要求的流程。

## 3. 分发形态（建议）

### 3.1 默认给用户的版本（推荐）

- `Lite 安装包（NSIS）`
- 不带本地模型和大依赖
- 首次启动按用户选择再安装

### 3.2 可选版本（后续）

- `Offline Pack / 离线包`（适合内网或教学机房）
- 可包含 Ollama/Whisper/部分模型
- 单独发布，不与默认安装包混在一起

## 4. 首次启动向导（用户流程）

### 4.1 第 1 步：欢迎页

展示：

- 软件说明（本地 AI / API / 混合模式都支持）
- 向导将帮助完成环境检查与模型配置
- 可稍后在 `设置 -> 首次向导` 重新打开（预留入口）

### 4.2 第 2 步：运行模式选择（必选）

用户选择其中一种：

1. `仅 API（推荐给新手）`
   - 不安装本地模型
   - 只配置云端 Provider（OpenAI-compatible / 其他）
   - 安装速度最快

2. `仅本地模型（离线优先）`
   - 安装/启动本地运行时（如 Ollama）
   - 勾选并下载本地模型
   - 不依赖云端 API

3. `混合模式（推荐进阶用户）`
   - 本地模型负责部分功能（如基础聊天、离线翻译）
   - API 负责更强模型/多模态/高质量生成

### 4.3 第 3 步：环境检查

根据模式检查：

- 应用内服务启动是否正常
- （如启用本地）本地运行时是否存在/可启动（如 Ollama）
- （如启用 API）是否已有 Provider 配置，或引导配置 Provider
- 网络连通性（仅 API 模式需要）

### 4.4 第 4 步：本地模型勾选（仅本地 / 混合模式）

支持多选，用户勾选想要的模型。

每个模型项显示：

- 模型名（含 `B` 规模）
- 推荐用途（RP / 学习 / 编码 / 通用）
- 推荐硬件（内存/显存）
- 是否启用 `Flash`
- 是否启用 `Thinking`

### 4.5 第 5 步：完成与写入配置

向导写入本地配置：

- `firstRunCompleted = true`
- `runtimeMode`
- 本地模型清单与模式映射
- 默认 Provider / 默认模型
- 语音、翻译、知识库等默认策略（可后续扩展）

## 5. 本地模型选择说明（给用户看清楚“几B”）

> 下面是向导里应展示给用户的“模型大小与建议”，避免用户盲选。

### 5.1 `B` 是什么意思？

- `7B / 8B / 13B / 14B / 34B` 里的 `B` 代表参数规模（十亿级参数）
- 一般来说：
  - 参数更大 => 理解/表达能力通常更强
  - 但也会更慢、更吃显存/内存

### 5.2 向导中的推荐模型（示例）

> 实际模型可按你项目支持情况扩展；下面是面向用户的默认建议列表。

| 模型 | 参数规模 | 典型用途 | 硬件建议（大致） | 备注 |
|---|---:|---|---|---|
| `qwen3:8b` | 8B | 通用/RP/学习 | 16GB 内存（CPU可用，较慢）或 6-8GB 显存 | 你项目默认本地首选之一 |
| `llama2:7b` | 7B | 基础聊天 | 16GB 内存或 6GB 显存 | 老牌稳定选项 |
| `llama2:13b` | 13B | 更稳的问答/写作 | 24GB+ 内存或 10-12GB 显存 | 比 7B 更慢 |
| `mistral:7b` | 7B | 通用/RP（速度好） | 16GB 内存或 6-8GB 显存 | 性价比较高 |
| `mixtral:8x7b` | MoE（重） | 高质量生成 | 高端机器（显存/内存要求高） | 不建议新手默认选 |
| `codellama:7b/13b/34b` | 7B~34B | 代码相关 | 看规模选择硬件 | 编码优先时再选 |
| `vicuna:7b/13b` | 7B~13B | 对话/RP | 中等硬件 | 版本差异大，需测试 |
| `yi:6b/9b/34b` | 6B~34B | 中文/双语（视版本） | 看规模选择硬件 | 具体效果看量化与版本 |
| `solar:10.7b` | 10.7B | 通用写作/对话 | 16-32GB 内存或 8-12GB 显存 | 质量不错、速度中等 |

### 5.3 给普通用户的选择建议（必须写清楚）

#### 如果用户完全不懂模型，默认推荐

1. 先选 `仅 API`（最快开始使用）
2. 或者本地模式只勾一个：`qwen3:8b` 或 `mistral:7b`
3. 先不勾大模型（如 `mixtral` / `34B`）

#### 如果用户电脑配置一般（笔记本 / 普通台式）

- 优先选 `7B/8B` 级别
- 不要一上来选 `13B+`
- 不要同时勾太多模型（占空间、下载慢）

#### 如果用户主要用于 RP（角色扮演）

- 先选 1-2 个对话模型
- 建议：`qwen3:8b` / `mistral:7b`
- 如果有高配机器，再尝试更大模型

#### 如果用户主要用于编程/代码解释

- 可以优先加 `codellama`
- 仍建议先从较小版本开始（如 `7b/13b`）

## 6. 每模型 `Flash / Thinking` 配置（你强调的重点）

### 6.1 规则（必须）

每个被勾选的本地模型都允许单独配置：

- `Flash`（快速模式）是否启用
- `Thinking`（深度模式）是否启用
- （可选）显示名称/别名
- （可选）默认参数模板（温度、max tokens 等）

### 6.2 不能再写死 Qwen 文案

如果用户没有启用 Qwen，本应用中任何地方都不应显示：

- `Qwen Flash`
- `Qwen Thinking`

统一改为模型无关文案（推荐）：

- `⚡ 快速模式（当前模型）`
- `🧠 深度模式（当前模型）`

示例：

- `⚡ 快速模式（qwen3:8b）`
- `🧠 深度模式（mistral:7b）`
- `⚡ 快速模式（DeepSeek-V3.2）`（API 模型同理）

### 6.3 用户如何选择 `Flash / Thinking`

给用户的简单说明（向导里直接显示）：

- `Flash`：更快，适合日常聊天、快速问答
- `Thinking`：更慢，但更适合复杂推理、长回答
- 如果你的电脑性能一般：先只开 `Flash`
- 如果模型不稳定或不支持思考风格：可以关闭该模型的 `Thinking`

## 7. API-only / 混合模式说明（给用户看的）

### 7.1 仅 API（推荐新手）

适合：

- 不想装本地模型
- 电脑性能一般
- 希望快速开始使用

说明：

- 需要配置 API Provider（例如 OpenAI-compatible）
- 不需要安装 Ollama / Whisper 本地模型（除非用户主动启用本地语音/本地 STT）

### 7.2 仅本地（离线优先）

适合：

- 希望离线可用
- 接受本地下载模型占空间
- 能接受较慢的 CPU 推理（或有独显）

### 7.3 混合模式（推荐进阶）

适合：

- 同时想要“本地可用 + 云端强模型”
- 需要按功能分别选 Provider（聊天 / 翻译 / 知识库 / 语音）

## 8. 实施建议（按阶段）

### Phase A（先做）

- 轻量安装包（不打包 `vendor/**/*` 大资源）
- 安装图标（使用 `图1.png` 转 `.ico`）
- 首次启动向导框架（页面 + 状态机）
- 模式选择（API-only / 本地 / 混合）
- 环境检查（本地运行时/API 可用性）

### Phase B（再做）

- 本地模型多选与下载
- 每模型 `Flash / Thinking` 配置
- 模型下拉文案全面改为“跟随当前模型”

### Phase C（打磨）

- 下载进度显示与失败重试
- “重新运行首次向导”入口
- 离线包（Offline Pack）单独分发方案

## 9. 验收标准（第一阶段）

满足以下条件即视为阶段完成：

- 用户能通过安装包完成安装并生成桌面图标
- 首次启动出现配置向导
- 用户可选择 `仅 API` / `仅本地` / `混合模式`
- 向导说明里清楚展示模型 `B` 规模和选择建议
- 完成向导后可进入主界面正常使用
- 不破坏现有功能：翻译 overlay / MCP / 知识库 / Lorebook / 语音等
